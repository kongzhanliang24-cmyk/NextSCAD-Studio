# 磅秤官网静态资源方案结构

---

# 一、整体架构

- 前端：Vue
  - 页面路由全部动态读取 JSON 数据
  - 视频资源放 NAS，提供 URL
- 后端：Spring Boot（後端先不實現）
  - 只提供 `contact` 表单接口
- 部署方式：
  - 内容更新 → 修改 JSON → 打包 Vue → 上传 dist → 重启/刷新

特点：

- 轻量
- 易部署
- 内容更新可直接改文件
- 不依赖数据库

# 四、Vue 路由结构

```js
const routes = [
  { path: '/', component: Home },

  { path: '/products', component: ProductList },
  { path: '/products/:categorySlug', component: ProductList },
  { path: '/product/:slug', component: ProductDetail },

  { path: '/solutions', component: SolutionList },
  { path: '/solutions/:slug', component: SolutionDetail },

  { path: '/cases', component: CaseList },
  { path: '/cases/:slug', component: CaseDetail },

  { path: '/about', component: About },
  { path: '/contact', component: Contact }
```

# 五、静态方案优势

- 不需要数据库

- 更新内容 → 直接改 JSON → 打包部署

- 可长期使用，适合少量产品和案例

- 后期如需动态，可轻松升级到数据库驱动

---

# 六、注意事项

1. JSON 文件目录建议按模块拆分，便于维护

2. 多语言建议使用嵌套结构 `{"zh": "...", "en": "..."}`

3. 视频尽量放 NAS 或对象存储，避免打包进前端

4. SEO 路由建议使用 history 模式，不要 hash
