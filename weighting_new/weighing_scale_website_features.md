## 网站整体定位

<details>
<summary>展开/收起</summary>

### 功能目标

| 功能       | 说明                                  |
| -------- | ----------------------------------- |
| 产品展示     | 展示所有磅秤产品型号，包含核心规格、应用场景、功能卖点和下载资料    |
| 解决方案展示   | 展示不同磅秤系统的解决方案，包括系统架构、核心功能、技术规格、客户价值 |
| 成功案例展示   | 展示客户使用磅秤系统的案例，包含项目背景、痛点、解决方案及效果     |
| 客户询价线索获取 | 提供产品/系统页面询价表单，收集客户信息用于销售跟进          |

**说明**：网站核心目标是“销售转化”，强调专业感、真实感、数据感和可信度。

</details>

## 首页功能模块

<details>
<summary>展开/收起</summary>

### 1️⃣ 顶部导航

**功能说明**：提供网站全局导航，用户可快速访问产品、系统、案例和公司信息，支持图文 Mega Menu。

| 功能   | 说明                         |
| ---- | -------------------------- |
| Logo | 返回首页                       |
| 磅秤   | hover 下拉展示分类和主打型号图片        |
| 磅秤系统 | hover 下拉展示系统列表和示意图         |
| 关于我们 | 静态链接到公司信息页                 |
| 语言切换 | 支持繁体 / English 切换，后续可扩展多语言 |

**技术实现**：使用 Vue + Mega Menu 组件，菜单内容动态读取 JSON 数据。

---

### 2️⃣ 轮播图（Hero Banner）

| 功能     | 说明                     |
| ------ | ---------------------- |
| 轮播图    | 3 张轮播图，支持标题、描述和 CTA 按钮 |
| CTA 按钮 | 链接至产品详情页或系统详情页         |

**技术实现**：可使用 `vue-awesome-swiper`，轮播图片和文案动态读取 JSON 配置。

---

### 3️⃣ 三大产品分类展示

| 功能   | 说明            |
| ---- | ------------- |
| 图片   | 产品示意图         |
| 名称   | 分类名称，如“防水工业秤” |
| 简短介绍 | 1-2 行描述       |
| 按钮   | 链接到该分类产品列表页   |

**布局建议**：使用 Flex/Grid 布局，支持 hover 动画。

---

### 4️⃣ 成功案例展示模块

| 功能       | 说明                 |
| -------- | ------------------ |
| 案例卡片     | 图片 + 项目名称 + 行业     |
| 点击跳转     | 跳转案例详情页            |
| Hover 效果 | 图片底部滑出系统名称、行业、成果数据 |

</details>

## 磅秤系统模块功能拆解

<details>
<summary>展开/收起</summary>

### 6.1 首屏

- 系统名称、子标题、CTA 按钮

### 6.2 客户痛点说明

- 人工记录易错、无法统计、无法对接 ERP

### 6.3 系统架构图

- 电子磅秤 → 仪表 → 工控机 → 管理软件 → 数据库 → ERP
- 扩展设备：条码枪、打印机、LED屏、车牌识别

### 6.4 系统组成

| 类型   | 功能                  |
| ---- | ------------------- |
| 硬件   | 磅秤、传感器、仪表           |
| 软件   | 数据采集、报表统计、权限管理、日志追踪 |
| 扩展设备 | 扫描枪、打印机             |

### 6.5 核心功能

- 自动称重记录、多用户权限、报表统计、ERP 对接

### 6.6 操作流程

货物上磅 → 系统自动采集 → 数据保存 → 打印标签 → 同步 ERP

### 6.7 技术规格

- 多磅接入、局域网/私有化部署、数据库备份、云扩展

### 6.8 客户价值

- 提升效率 30%、降低人工成本、避免数据错误、实现可追溯管理

## 静态资源与部署方案

<details>
<summary>展开/收起</summary>

- JSON 数据管理，无数据库依赖，轻量级

- 目录结构清晰：
  
  ```
  assets/images/products
  assets/images/solutions
  assets/images/cases
  data/products
  data/solutions
  data/cases
  ```

- 内容更新流程：修改 JSON → 打包 Vue → 上传 dist → 刷新

- 后期可升级为数据库驱动模式

</details>
